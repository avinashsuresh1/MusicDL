🎼 MusicDL Schema Reference

MusicDL is a folder-structured, YAML-based Music Definition Language designed for composing music using relative pitch intervals and additive synthesis.

Project File Structure

Every project resides in a single folder containing a metadata file and subfolders:

my-project/ ├── composition.yaml ├── instruments/ │ ├── pad.yaml │ └── bass_synth.yaml ├── melodies/ │ └── theme.yaml ├── chords/ │ └── c_major.yaml └── tracks/ └── lead_track.yaml

composition.yaml Schema

Defines global composition metadata and microtuning rules:

title: "Moonlit Sonata" # Composition title (non-empty string) tempo: 90 # Playback tempo in beats per minute (BPM) root_frequency: 261.63 # Frequency in Hz of pitch 0 (Middle C) interval: 100 # Size of one pitch step in cents (default 100)

Relative Pitch Formula

Notes define pitch in integer intervals w.r.t the root pitch 0. The frequency of a note is calculated as:

f = froot × 2(pitch × interval) / 1200
  • By default, interval: 100 (equal temperament semitones). Pitch 12 is an octave higher.
  • To use quarter-tones (24-TET), set interval: 50.

Instruments define timbres using additive synthesis (adding sine waves at different frequencies) and an optional ADSR volume envelope.

Harmonic Spectrum Schema

Each instrument YAML file in instruments/ contains a list of partials:

# instruments/sine_pad.yaml harmonics: - { z: 1, amplitude: 1.0 } # Fundamental frequency (pitch frequency) - { z: 2, amplitude: 0.5 } # 2nd harmonic (octave higher) - { z: 3, amplitude: 0.25 } # 3rd harmonic (perfect fifth) - { z: 0.5, amplitude: 0.3 } # Subharmonic (octave lower, non-integer z)
  • z: Frequency multiplier (positive number, can be non-integer for inharmonic timbres).
  • amplitude: Level of this partial (float between 0.0 and 1.0).

ADSR Envelope Schema

Specifies how the volume of the note shapes over time (optional, defaults to a fast 10ms attack and 50ms release):

# instruments/sine_pad.yaml (continued) adsr: attack: 150 # Attack duration in milliseconds (fade in) decay: 200 # Decay duration in milliseconds (ramp to sustain level) sustain: 0.6 # Sustain volume level (float, 0.0 - 1.0) release: 600 # Release duration in milliseconds (fade out after release)

Melodies and Chords contain the note data, and Tracks mix and sequence them on the timeline.

Melody Schema

Melodies define a sequence of notes. Each file in melodies/ contains sequential note data (usually relative to start beat 0):

# melodies/theme.yaml instrument: sine_pad # References an instrument name in instruments/ notes: - { pitch: 0, offset: 0.0, duration: 1.5 } # Root note - { pitch: 4, offset: 1.5, duration: 0.5 } # Next note - { pitch: rest, offset: 2.0, duration: 1.0 } # Silence / Rest
  • pitch: Integer interval from root frequency, or the string 'rest'.
  • offset: Relative starting beat position (float >= 0).
  • duration: Note duration in beats (float > 0).

Chord Schema

Chords define pitches played simultaneously. Because they are simultaneous, chords do not need offsets or durations inside their YAML file. Each file in chords/ contains:

# chords/c_major.yaml instrument: choir_pad # References instrument in instruments/ pitches: [0, 4, 7] # Pitch intervals played together

Track Schema

Tracks sequence melodies and chords. You can play a melody or chord at any point on the timeline by specifying its offset (and duration for chords) in the track YAML file in tracks/:

# tracks/main_track.yaml volume: 0.8 # Master track volume level (float, 0.0 - 1.0) melodies: # Play the theme melody twice at different offsets - { name: theme, offset: 0 } - { name: theme, offset: 6 } chords: # Play C Major chord at beat 0 and beat 3 for 3.0 beats duration each - { name: c_major, offset: 0, duration: 3.0 } - { name: c_major, offset: 3, duration: 3.0 }
